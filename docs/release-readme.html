Oblivion Reverse

Release Readme

This project aims to reconstruct the original Oblivion Quake II mod source code as accurately as possible, including original retail bugs and quirks.

The reconstruction is now in a mature, playable state from limited testing, but it should still be understood as a retail-parity recovery effort rather than a redesigned remaster.

This release archive contains an oblivion/ directory with:

  • the platform-specific game module binary
  • oblivion.cfg
  • this README.html document

Installation

  1. Install the original Oblivion mod first from the original Oblivion release page.
  2. Locate the Quake II installation that already contains the installed oblivion/ game directory.
  3. Extract this release archive directly into that Quake II installation.
  4. Allow the extraction to replace the existing platform binary and oblivion.cfg.
  5. Launch Quake II with +set game oblivion.

This archive is an update for an existing Oblivion installation, not a standalone distribution of the full mod data.

Included Binary Names

The original retail Oblivion release was Windows-only. This reconstruction provides replacement game-module binaries for Windows, Linux, and macOS.

Platform Binary
Windows gamex86.dll
Linux game.so
macOS game.dylib

Project Intent

Oblivion Reverse is a retail-parity reconstruction project. Its purpose is preservation and accurate source recovery, not redesign. If retail Oblivion behaved strangely, the reconstruction often needs to behave strangely too.

The cross-platform support in this archive comes from the reconstruction project itself; it is not a property of the original retail release.

Versioning

Nightly builds use the repository semantic base version plus a UTC nightly stamp, for example: v1.0.0-nightly.20260316
